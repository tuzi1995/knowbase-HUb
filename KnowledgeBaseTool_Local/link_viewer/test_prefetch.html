🚀 分页预加载功能测试指南

📋 功能状态

✓ 已实现 实现日期: 2026-04-22

  • 智能预加载下一页数据
  • LRU缓存策略（最多10页）
  • 5分钟缓存过期机制
  • 自动缓存清除（筛选/搜索/排序时）
  • 静默失败处理（不影响主流程）

🧪 测试步骤

测试1：基本预加载功能

  1. 打开主页面：index.html
  2. 切换到"知识库"标签页
  3. 按 F12 打开浏览器控制台
  4. 等待第1页数据加载完成
  5. 观察控制台，应该看到：✓ 预加载第 2 页成功
  6. 点击"下一页"按钮
预期结果：
  • 页面立即显示第2页数据（无"加载中..."提示）
  • 控制台显示：✓ 使用缓存数据（第 2 页）
  • 控制台显示：✓ 预加载第 3 页成功

测试2：缓存清除机制

  1. 在第2页的状态下
  2. 在"搜索产品名称"输入框中输入任意文字
  3. 点击"🔍 搜索"按钮
  4. 等待搜索结果加载
  5. 点击"下一页"
预期结果：
  • 显示"加载中..."（因为缓存已清除）
  • 加载完成后，再次预加载下一页

测试3：排序改变

  1. 点击任意列标题进行排序（如"ID"列）
  2. 观察控制台
  3. 点击"下一页"
预期结果：
  • 排序改变后，缓存被清除
  • 下一页需要重新加载
  • 新的排序下，预加载正常工作

测试4：页面大小改变

  1. 找到"每页显示"下拉框
  2. 将50改为100
  3. 观察页面重新加载
  4. 点击"下一页"
预期结果：
  • 页面大小改变后，缓存被清除
  • 使用新的页面大小进行预加载

测试5：状态筛选

  1. 点击"未调整"、"修改中"或"删除中"状态标签
  2. 观察数据筛选
  3. 点击"下一页"
预期结果：
  • 状态筛选改变后，缓存被清除
  • 筛选后的数据正常预加载

测试6：表格切换

  1. 点击"前刻库"单选按钮
  2. 观察数据切换
  3. 点击"下一页"
预期结果：
  • 表格切换后，缓存被清除
  • 新表格的数据正常预加载

🔍 调试信息

在浏览器控制台（F12 → Console）中可以看到以下日志：

  • ✓ 使用缓存数据（第 X 页） - 缓存命中，直接使用缓存
  • ✓ 预加载第 X 页成功 - 后台预加载成功
  • 预加载失败（不影响功能）: ... - 预加载失败（不影响主流程）
💡 提示： 如果看不到这些日志，请确保控制台的日志级别设置为"详细"或"全部"。

⚠️ 注意事项

重要：
  • 预加载功能不会影响任何现有功能
  • 预加载失败时会静默处理，不会显示错误
  • 缓存会在5分钟后自动过期
  • 最多缓存10页数据，超出后自动清理最旧的
