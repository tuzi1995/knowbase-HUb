 K-Matrix 助手 - 产品使用说明书

            
        

        

            
            
            

                
 K-Matrix 助手 (K-Matrix Assistant)

                

                    定位 ：一站式智能知识库管理与质量评估平台，集成了数据维护、机型矩阵配置、多媒体资源管理及大模型自动化评分治理功能。
                

            


            
            

                
 1. 知识库管理 (Knowledge Base Management)

                
核心数据维护模块，支持对主库 (V1) 和测试库 (V1T-1) 的全生命周期管理。

                
                

                    

                        
 数据导入 (Data Import)

                        
    
                            
  • 支持格式 ：Excel (.xlsx, .xls)、CSV。
    • 
                            
  • 三种导入模式 ：
    • 
                            
      
	                                
    • 增量更新 (Upsert) ID 存在则更新，不存在则新增（推荐用于日常维护）。
      • 
	                                
    • 批量新增 (Append) 自动跳过已存在的 ID，仅插入新数据（安全补充）。
      • 
	                                
    • 全量覆盖 (Overwrite) 导入前先自动将 V1 同步至 V1T-1 备份，再清空 V1 并重新导入（用于重大版本重置）。
      • 
	                                
    • 同步删除缺失 ID 可选开关；V1 中存在但导入文件 ID 列未包含的记录，会二次确认后从 V1 物理删除。
      • 
	                            
    
                            
  • 辅助功能 ：
    • 
                            
      
                                
    • 查重预览 ：导入前检查重复 ID，避免冲突。
      • 
                                
    • 模板下载 ：提供标准 Excel 导入模板。
      • 
                            
    
                        

                    

                    

                        
 数据同步 (Data Sync)

                        
    
                            
  • 同步此刻到前刻 ：将 V1 库（此刻状态）全量覆盖同步至 V1T-1 库（前刻备份）。
    • 
                            
  • 一键同步下游 ：将当前 V1 同步至机型矩阵、多媒体预览、评分表；治理月度召回仍须在治理 Tab 导入 Excel，本操作会更新评分快照供治理关联展示。
    • 
                        

                    

                    

                        
 检索与筛选 (Search & Filter)

                        
    
                            
  • 多维搜索 ：支持按 ID、产品名称、问题、答案、URL 进行模糊匹配。
    • 
                            
  • 状态筛选 ：可按“未调整”、“修改中”、“删除中”三种状态快速过滤记录。
    • 
                        

                    

                    

                        
 编辑与操作 (Edit & Actions)

                        
    
                            
  • 在线编辑 ：弹窗式编辑，支持 Markdown 实时预览，快捷勾选关联机型。
    • 
                            
  • 批量操作 ：支持批量删除、批量重置修订状态（完成本轮修订）。
    • 
                            
  • 型号库管理 ：内置产品型号管理器，支持动态增删产品分类及具体型号。
    • 
                            
  • 数据导出 ：支持导出当前筛选结果或全量数据。
    • 
                        

                    

                

            


            
            

                
 2. 机型矩阵管理 (Product Matrix)

                
专门用于维护“问题-机型”对应关系的二维矩阵视图。

                

                    

                        
 矩阵配置

                        
    
                            
  • 矩阵配置 ：直观展示每个问题适用的机型，支持单元格点击切换（适用/不适用）。
    • 
                            
  • 批量配置 ：支持整列（某机型）或整行（某问题）的批量开启/关闭。
    • 
                            
  • 列操作 ：支持动态添加新机型列、复制现有列配置。
    • 
                            
  • 同步逻辑 ：将矩阵中的配置关系自动同步回知识库的 product_name 字段，确保数据一致性。
    • 
                            
  • 导入导出 ：支持矩阵结构的独立导入与导出。
    • 
                            
  • 四个常用按钮（白话）：
                                
      
                                    
    • ✅ 设为已配置：先勾选多行问题 → 选一个机型列 → 把这些格子统一标成“对该机型生效”。例：勾选 10 条“尘袋/滤网”问题，选择机型 G30S Pro，批量设为✅。
      • 
                                    
    • 🚫 设为未配置：先勾选多行问题 → 选一个机型列 → 把这些格子统一标成“对该机型不生效”。例：把“不带蒸汽功能”的机型，在“蒸汽护理”相关问题上统一设为❌。
      • 
                                    
    • 提交修改：把你刚才单格点选/批量设置产生的“待提交修改”一次性正式保存，并写入提交日志与修改历史。例：你刚批量改了 100 个格子，不点提交刷新会丢；点提交后才算落库。
      • 
                                    
    • 提交日志：查看每次提交是否成功、失败原因、重试次数（按 operation_id 追踪）。例：提交后担心 Supabase 写入是否成功，打开提交日志就能看到 success/failed 与错误信息。
      • 
                                
    
                            
    • 
                        

                    

                    

                        
 筛选逻辑（行筛选 vs 表头收缩）

                        
    
                            
  • 固定列：表格左侧固定展示 ID / 问题 / 答案，右侧为各机型列。
    • 
                            
  • 搜索问题：只筛选数据行（哪些问题显示），不影响表头机型列。
    • 
                            
  • 搜索机型：只筛选数据行（只显示包含该机型的记录），不收缩表头机型列。
    • 
                            
  • 产品分类：同时筛选数据行 + 收缩表头机型列（只保留该分类机型）。
    • 
                            
  • 映射分类：筛选数据行（完全匹配：该问题关联机型集合必须与该映射机型集合一致，顺序不要求一致）+ 收缩表头机型列（只保留该映射对应机型）。
    • 
                            
  • 表头型号(多选)：仅收缩表头机型列，不筛选数据行；与“产品分类/映射分类”叠加时取交集。
    • 
                            
  • 列收缩合并规则：最终表头机型 = 产品分类白名单 ∩ 映射分类白名单 ∩ 表头型号多选白名单（未选的条件不参与）。
    • 
                            
  • 导出矩阵：当前仅跟随“搜索问题/搜索机型”的行筛选导出；表头收缩与分类 chips 不影响导出列。
    • 
                        

                    

                

            


            
            

                
 3. 多媒体预览 (Multimedia Preview)

                
集中管理知识库中引用的图片、视频及链接资源。

                

                    

                        
 资源管理与预览

                        
    
                            
  • 资源聚合 ：自动提取知识库中的所有 URL 链接。
    • 
                            
  • 在线预览 ：
    • 
                            
      
                                
    • 图片 ：支持点击放大查看细节。
      • 
                                
    • 视频 ：支持直接在页面内播放。
      • 
                            
    
                            
  • 标签管理 ：支持批量为资源打标（如 image , video , manual ）。
    • 
                            
  • 反向追溯 ：点击资源 ID 可直接跳转至知识库管理页面的对应记录，方便修改上下文。
    • 
                        

                    

                

            


            
            

                
 4. 知识库评分 (Scoring)

                
利用大语言模型 (LLM) 对问答质量进行多维度自动化评估。

                

                    

                        
 评分执行

                        
    
                            
  • 随机抽样 ：支持快速抽取 10/50/100 条数据进行验证。
    • 
                            
  • 全量评分 ：对整个知识库进行批量打分。
    • 
                            
  • 同步数据 ：从 V1 库同步最新待评分数据。
    • 
                        

                    

                    

                        
 评分维度

                        
    
                            
  • 质量 (Quality) 准确性、完整性。
    • 
                            
  • 合规 (Compliance) 是否符合品牌规范。
    • 
                            
  • 时效 (Timeliness) 信息是否过时。
    • 
                            
  • 解决力 (Resolution) 能否有效解决用户问题。
    • 
                            
  • 非冗余 (Non-redundancy) 是否存在重复信息。
    • 
                            
  • 多媒体 (Multimedia) 图文/视频加分项。
    • 
                        

                    

                    

                        
 配置管理

                        
    
                            
  • API 配置 ：自定义 LLM 的 API Key、Base URL 和模型名称。
    • 
                            
  • 标准配置 ：支持自定义 System Prompt（评分标准）。
    • 
                            
  • 报告导出 ：支持导出详细的评分报告及统计结果。
    • 
                        

                    

                

            


            
            

                
 5. 治理与记录 (Governance & Logs)

                

                    

                        
 知识库治理 (Governance)

                        
用于数据的定期回顾与生命周期管理。

                        
    
                            
  • 按月召回 ：查看特定月份的历史数据召回情况。
    • 
                            
  • 清理维护 ：支持对过期或无效的召回记录进行批量删除。
    • 
                        

                    

                    

                        
 修改记录 (Modifications)

                        
系统的审计日志模块。

                        
    
                            
  • 日志查看 ：记录所有关键操作（新增、编辑、删除、矩阵变更）。
    • 
                            
  • 详情追踪 ：展示操作时间、操作人、变更类型及具体变更内容。
